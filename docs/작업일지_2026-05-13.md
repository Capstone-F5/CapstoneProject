## 1. 주문내역(CartScreen) 카드 레이아웃 전면 개편

### 기존 구조
- 3열 그리드 (`1fr / COL_QTY / COL_PRICE`)
- 이미지·상품명·옵션박스가 한 열에 혼재

### 변경 구조
4열 그리드로 전환: `IMG_SIZE(90px) / 1fr / COL_QTY(130px) / COL_PRICE(140px)`

| 위치 | 내용 |
|------|------|
| Row 1+2, Col 1 | 이미지 + 상품명 (전체 높이 기준 세로 중앙) |
| Row 1, Col 3 | 수량 조절 버튼 |
| Row 1, Col 4 | 금액 + 삭제(✕) |
| Row 2, Col 2~4 | 세부 옵션 회색 박스 |

- 세부 항목 없는 경우: 수량·금액 셀이 `gridRow: '1/3'`으로 양 행 전체에서 세로 중앙 정렬
- 이미지+상품명 컨테이너는 항상 `gridRow: '1/3'`으로 전체 높이의 중앙에 위치

### 컬럼 헤더 정렬
- 헤더 그리드도 동일한 4열 구조 적용
- "메뉴" → col 1 중앙, "수량" → col 3 중앙, "금액" → col 4 중앙

### 세부 옵션(SubRow) 컬럼 정렬
- `gridTemplateColumns: 1fr / COL_QTY / COL_PRICE`로 메인 그리드와 폭 일치
- `columnGap: 12`로 헤더·카드 그리드와 수직 정렬
- 금액 열 `paddingRight: 26`으로 헤더 "금액" 텍스트와 정렬

## 3. 결제 흐름 전체 팝업화

기존에는 결제 관련 화면들이 별도 페이지(`nav('payment')`, `nav('cardPayment')` 등)로 전환됐으나, 장바구니 화면 위에 팝업으로 뜨도록 전환.

### 팝업 흐름

```
결제하기 버튼
  └─ 포인트 적립 여부 팝업
       ├─ 아니오 → 결제 수단 선택 팝업
       └─ 예 → 전화번호 입력 팝업
                 └─ 확인 → 결제 수단 선택 팝업
                            ├─ 신용카드/삼성페이 → 카드 결제 대기 팝업
                            ├─ 현금 → 현금 결제 대기 팝업
                            └─ 간편결제 → 바코드 스캔 팝업
```

- 각 결제 대기 팝업에서 취소 시 → 결제 수단 선택 팝업으로 복귀
- 카드 결제 완료 시 → 장바구니 초기화 후 시작 화면으로 이동

## 4. 결제 수단 선택 팝업 UI

### 구조

```
안녕하세요, OOO님       ← 포인트 번호 입력 시에만 표시
결제 수단을 선택하세요
XX,XXX원

[ 신용카드 / 삼성페이 ] [ 현금 ]   ← 2×1 그리드, 각각 독립 버튼

간편결제
[ 네이버페이 ] [ 카카오페이 ]       ← 2×2 그리드
[ 제로페이   ] [ 페이코     ]

[ 취소 ]
```

- 신용카드+삼성페이는 하나의 버튼으로 묶어 좌측, 현금은 우측
- 간편결제 각 항목: `logo` 필드가 `null`이면 텍스트 폴백, 이미지 파일 추가 시 자동 전환
